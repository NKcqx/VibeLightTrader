# Vibe Coder 复刻指南

> 写给"用 Cursor / Claude Code / Codex 这类 coding agent 协作、自己不一定写代码、想从 0 复刻一个类似项目"的人。
>
> **这不是技术文档。** 技术怎么实现写在 [`../tutorial/`](../tutorial/) 里。这份文档讲的是 **过程**——我怎么跟 AI 一起把这个项目从一句话需求做到今天的样子，以及你要复刻一个类似的，可以怎么走。

---

## 这份指南想解决什么

vibe coder 复刻项目时，最大的痛点不是"看不懂代码"——AI 能讲。痛点是：

- ❓ 我该跟 AI 说什么 / 不该说什么？
- ❓ 看见 AI 跑偏了，我该什么时候打断？
- ❓ 哪些事我必须自己干、AI 替不了？
- ❓ 一个项目应该怎么循序渐进，不要一口气吞？
- ❓ 我没那么多代码功底，怎么知道现在该往哪走？

这份文档的每一节都直接打这些痛点。

## 为什么是这个项目

`equity-monitor` 是一个 **多空仓中长线美股监控 + 模拟交易** 系统：

- 数据：富途 OpenD（免费）
- 调度：每小时拉行情、算 RSI/MACD/Boll、检测信号
- 决策：用 LLM（cursor-agent）按"投资者画像"出 BUY/SELL/HOLD
- 执行：模拟下单到富途 SIMULATE 账户（手机/网页都能看持仓）
- 通知：飞书卡片
- 观测：每标的一份 Markdown journal + 决策审计日志

这个项目的几个**复刻价值高**的点：

1. ✅ **零 LLM API 成本** —— 用 cursor-agent 复用 IDE 订阅
2. ✅ **零量化数据成本** —— 富途免费 API
3. ✅ **持仓外部可见** —— 模拟账户在富途 App 直接看
4. ✅ **配置驱动 + Protocol 抽象** —— 换数据源 / 换 LLM / 换通知 = 改一行 yaml
5. ✅ **可循序渐进** —— 7 天的剧本可以从最简版一直长到全自动

如果你有相似需求（盯盘 / 自动决策 / 集成订阅 LLM / 飞书通知），这份指南能省你 1-2 周的弯路。

## 你需要的资源（最低版）

| 类型 | 必需 | 替代品 |
|---|---|---|
| Coding agent | Cursor（推荐） | Claude Code / Codex CLI / Aider |
| LLM 订阅 | Cursor Pro/Max | Claude Pro / OpenAI Plus / 自有 API key |
| 数据源 | 富途 OpenD（自动接通模拟账户）| Yahoo / IBKR / Bloomberg |
| 通知通道 | 飞书机器人 | 邮件 / Slack / 微信公众号 |
| Python 环境 | 3.11+ | 3.10 也能跑 |

如果以上**至少 3 项你已经有了**，复刻成本就很低。
