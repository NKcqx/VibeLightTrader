# Part A · 立项与心智设定

## 那句话其实是后来补出来的

我现在重述这个项目的目标，听起来很顺：

> 5 万美金 / 标的，3-6 个月持有，可接受 ±20% 浮动，标的 NVDA + MSFT，AI 替我盯盘出 BUY/SELL，下到富途模拟账户。

但**我开局并没有这么具体**。我最早那条消息只说了"做一个股票监控和模拟操盘的功能、每小时监控价格、做技术面分析、最好有基本面"——很泛。具体的金额、周期、回撤容忍度、标的，都是中间陆陆续续补出来的：

- "每个标的 5W 美金、上下浮动 20%" 是中段我说要做策略时才提的；
- "NVDA + MSFT 两只" 是更后期为了减少并行复杂度才框死的；
- "可接受 LLM 自动下单" 是 LLM 第一次给 BUY 之后我才确认的；
- 投资者画像里 21 个字段，是 LLM 满仓买完我才反应过来需要的。

所以这一节真正的意思**不是**"开局必须写出完美的一句话"，而是：

- 你心里有一团模糊的目标，没关系
- **每次 AI 的输出让你不踏实**——那一刻你脑子里**多出了一条隐性偏好**——把它说出来，加进配置 / spec / prompt
- 项目的"宪法"是这样一段段长出来的，不是一上来就完整

唯一别做的是：用"我想做个量化系统"这种大词蒙混。AI 听到大词只会还你一个大而无当的脚手架。

## 资源选型：vibe coder 真正动脑子的地方

vibe coder 不写代码，所以选型这一关你逃不掉。AI 知道有哪些选项，但**它不知道你已经有什么、不知道你能接受多少额外成本**——这一关只能你自己拍板。

我开局就把已有资源贴了过去（"我推荐这个富途牛牛的 SKILL，看下能否用上"），AI 评估完，技术栈基本就定了一半。后面再补 LLM 和数据库。把当时的思考过程拆成 4 列：

| 资源 | 我的需求 | 可选方案 | 最终选择 & 原因 |
|---|---|---|---|
| **行情数据** | 实时报价 + 历史 K 线 + RSI/MACD 指标 + **要能模拟下单** | yfinance / Alpha Vantage / IBKR API / Alpaca / Bloomberg / **富途 OpenAPI** | 富途 OpenAPI（OpenD）。**我已有富途账号**，免费、覆盖中港美，自带模拟账户能在富途 App 直接看持仓——省了一大块持仓 UI 的活。弯路：我开局贴了富途 SKILL 链接，但 AI 仍习惯性问要不要 fallback yfinance，我后来明确说"先把富途吃透"才不分心 |
| **LLM 推理** | 出 BUY/SELL 决策；**月增量成本 ≈ 0**；中长线推理深度要够 | Anthropic API（贵）/ OpenAI API（贵）/ DeepSeek（便宜但金融弱）/ Ollama 本地（强模型跑不动）/ **cursor-agent CLI**（小众）/ Claude Code SDK（同思路） | cursor-agent CLI——把 Cursor Pro 订阅反向当 API 用。这是个**小众选择**：CLI 启动慢（30-60s）、不能控温度、并发少。但单符号决策一天就几次，慢一点无所谓；订阅已经付过了，月增量 $0。先走过 HITL（在 IDE 复制粘贴）发现确实慢，AI 提了一句"Cursor 有 CLI 吗"，我去查到了 cursor-agent，直接跳档自动化 |
| **通知** | 手机能收 + 富文本卡片 + 公司就在用 | 邮件 / Slack / Telegram / 微信公众号 / **飞书机器人** | 飞书 + lark-cli。公司已用、有官方 CLI 工具、卡片漂亮。**绕过的弯**：原本想做双向（飞书发命令控制系统）卡了一晚没成（参见 [Part C](./C-evolution-timeline.md) 第 6 节）。最后用 CLI 命令替代了——绕过去比怼上去聪明 |
| **模拟账户** | 能下单、能查持仓，**最好不用我自己写持仓 UI** | 自建持仓表 + 自己写网页 / Alpaca paper / **富途模拟账户** | 富途模拟账户。复用富途账号、富途 App 直接看持仓和盈亏——**省了一周的 UI 开发** |
| **数据库** | 单机、零运维、能跨进程读 | SQLite / PostgreSQL / DuckDB / Parquet 文件 | SQLite。一个人用，AI 默认建议 Postgres 我让它改了 |
| **Python 环境** | 隔离、能跑 numpy/pandas | venv / **conda** / uv / poetry | conda env `fin`。我本机已有 conda；AI 几次自作主张装 uv 我都打断了——保留单一工具链让协作简单 |
| **Coding agent** | 能在 IDE 里跟 LLM 协作 | Cursor / Claude Code / Codex CLI / Aider | Cursor。已订阅，且后面才发现可以反向接 cursor-agent，**一笔订阅当两笔用** |

几个能泛化的姿势：

1. **先列你已有的，再问 AI 够不够用**——这比让 AI 从空白处推荐快一个量级
2. **小众路径常常更划算**——cursor-agent 在网上几乎没人写，但因为我已经有 Cursor 订阅，它就是这个项目最大的成本杠杆
3. **先选一个跑通再优化**——富途、SQLite、conda 这些都不是"最优"，是"最近"。能跑就先跑

> ⚠️ 老实说：这张表是回头复盘的样子。**当下我并没有这么清醒地比较过**——很多决策是 AI 列了 3 条我直接挑了顺眼的，事后证明对了；也有几次挑错了又改回来（开发数据库一开始用了 PG 的 schema，跑了一圈发现 overkill 又退到 SQLite）。这张表的价值不是"照我这样选"，而是**告诉你哪些维度需要你自己拍板**。

## 别一上来就要 🟢

| 档 | 长什么样 |
|---|---|
| 🔴 阈值告警 | 每小时拉行情，触发阈值就推一张图。决策全靠你 |
| 🟡 LLM 出建议 + 你点确认 | 飞书卡片附 BUY/SELL 推荐，你按按钮才下单 |
| 🟢 全自动 | LLM 直接下到模拟账户，你只看消息 |

我的实际路径是 🔴 → 🟡（HITL）→ 🟢（cursor-agent）。每档跑了几天才升级。

直接奔 🟢 的风险不是技术，是**没给自己留时间确认 LLM 判断的质量**。我后来第一次看到 LLM 满仓 251 股 NVDA 的 BUY，第一反应是不踏实——那种不踏实只能在 🟡 档建立信心后才会消失。

## 角色边界

vibe coder 容易自以为是"用户"——AI 写代码、我用就行。实际不是。

| 我做的 | AI 做的 |
|---|---|
| 决定一句话目标、改 spec、否决新功能 | 把目标翻成代码、写 spec、列方案 |
| 选型时拍板（特别是用已有资源） | 列 trade-off |
| 把不踏实变成显性配置 | 写 prompt 和错误处理 |
| 知道飞书后台 / 富途 App / 本机 cwd 长什么样 | 假设标准环境，看不到你看到的 |
