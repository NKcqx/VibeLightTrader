# Part A · 立项与心智设定

## 你那句话写得够具体吗

我开局那条信息差不多是这样：

> 5 万美金 / 标的，3-6 个月持有，可接受 ±20% 浮动，标的固定 NVDA + MSFT，AI 替我盯盘 + 出 BUY/SELL/HOLD 建议，最终下单到富途模拟账户。

这一句话后面**所有的代码改动都从它推出来**。投资者画像、模拟账户接口、cursor-agent 选型、调度时段——每一项都能 trace 回这一句。

写不出来这种具体度的句子，先别 clone 任何 repo。**"我想做一个量化系统"不是项目目标**，AI 听到这种大词只会回你一个大而无当的脚手架。

---

## 你已经有的，比 AI 能选的更重要

我开局就贴了富途牛牛的官方 SKILL 链接，说"看下能否用上"。一句话，AI 几分钟评估完，整个数据栈就定了。

如果我反过来问"做股票盯盘有哪些方案"，AI 会列十几条从 yfinance 到 Bloomberg 的 trade-off，我会淹死在选型里。

最后这个项目用到的栈——

| 资源 | 我已经有的 |
|---|---|
| 数据 | 富途账号 + OpenD（自带模拟交易） |
| LLM | Cursor Pro 订阅（后来发现可以用 cursor-agent CLI 反向当 API 用） |
| 通知 | 飞书机器人 + lark-cli |
| Coding agent | Cursor IDE |
| Python | conda env `fin` |

——开局那条消息里全部出现过。**AI 没替我选栈，我替自己选了**，AI 只是判断了可行性。

> **可借鉴**：开局之前列一份你的资源清单（订阅、账号、内部工具、SKILL 包），一次性扔过去，比让 AI 从零推荐快 10 倍。后面对话里 AI 也会优先在这堆里挑实现。

## 别一上来就要 🟢

我最终落在 🟢 全自动，但**不是一开始就 🟢**。

| 档 | 长什么样 |
|---|---|
| 🔴 阈值告警 | 每小时拉行情，触发阈值就推一张图。决策全靠你 |
| 🟡 LLM 出建议 + 你点确认 | 飞书卡片附 BUY/SELL 推荐，你按按钮才下单 |
| 🟢 全自动 | LLM 直接下到模拟账户，你只看消息 |

我的实际路径是 🔴 → 🟡（HITL：LLM 在 Cursor IDE 里出建议，我贴回脚本）→ 🟢（cursor-agent CLI 接管）。中间每一档都跑通了几天才升下一档。

直接奔 🟢 的最大风险**不是技术**，是没有给自己留时间确认 LLM 判断的质量。我后来在 7 节看到 LLM 给出"满仓 251 股"的 BUY，第一反应是不踏实——那种不踏实只能在 🟡 档建立信心后才会消失。

## 角色边界

vibe coder 容易自以为是"用户"——AI 写代码、我用就行。实际不是。

| 我做的 | AI 做的 |
|---|---|
| 决定一句话目标、改 spec、否决新功能 | 把目标翻成代码、写 spec、列方案 |
| 看到跑偏立刻打断 | 把它"觉得对"的实现写下去 |
| 把不踏实变成显性配置（投资者画像 21 个字段就是这么来的） | 提示词、prompt、错误处理 |
| 知道飞书后台 / 富途客户端 / 本机 cwd 长什么样 | 假设标准环境，看不到你看到的 |

**反直觉点**：你越被动，AI 越容易越界、项目越容易烂尾。vibe coder 不是省心的工种，是**比工程师更主动地拍板**的工种。
